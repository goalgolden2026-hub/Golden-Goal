# Contributing to Golden Goal

We welcome contributions to the Golden Goal predictive forecasting platform! Whether you're fixing bugs, improving docs, or writing smart contracts, we appreciate your help.

## Code of Conduct

By participating, you agree to uphold our professional standards:
- Be respectful and cooperative.
- Focus on constructive feedback and improvements.

## How to Contribute

1. **Fork the Repository**: Create a fork of the codebase.
2. **Create a Feature Branch**: Use descriptive branch names: `git checkout -b feature/your-feature-name`.
3. **Write Type-Safe Code**: If modifying TypeScript components or server modules, ensure type validation passes cleanly.
4. **Anchor Program Rules**: When adding Anchor/Rust code, ensure all tests in the `/program` directory pass successfully.
5. **Open a Pull Request**: Submit a detailed PR description explaining the changes.

## Testing Guidelines

Before opening a PR, always check that:
- Code compiles flawlessly: `npm run build` passes with zero warnings.
- There are no syntax or linter errors.

Thank you for contributing to the future of Solana prediction economies!
